# Icon Guidelines

## Conventions

- Use icons to reinforce meaning, not as the only visual cue.
- Keep icons simple, consistent, and monochrome where possible.
- Use inline SVG or icon components inside card headings, legend keys, and status badges.

## Size

- Use 16px to 18px icons for label and status elements.
- Use 24px to 32px icons for section headings and callouts.
- Use larger illustrations only when they support a clear visual hierarchy.

## Spacing

- Space icons from text with a small margin or `gap` utility.
- Align icons vertically center with text labels.
- Keep icon spacing consistent across cards and navigation items.

## Categories

- Navigation icons for lists and poster entry points.
- Status icons for semantic labels such as success, warning, and info.
- Concept icons for architecture domains, flows and systems.
