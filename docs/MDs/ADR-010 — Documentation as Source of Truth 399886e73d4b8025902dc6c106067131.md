# ADR-010 — Documentation as Source of Truth

---

**Status:** Accepted

### Context

A medida que el proyecto crece, mantener múltiples versiones de los documentos aumenta el riesgo de inconsistencias.

### Decision

Cada documento tendrá una única versión vigente.

Las decisiones arquitectónicas se registrarán mediante ADRs y el estado del proyecto se reflejará en Project State.

### Consequences

- La documentación permanece consistente.
- Se evita la duplicación de documentos.
- La evolución de la arquitectura queda registrada sin perder claridad.
- El POS adopta un enfoque **Docs as Code**.
